## 1. 游戏基本功能需求

### 1.1 核心玩法功能

#### 1.1.1 蛇的移动控制
- 蛇在20×20的网格上移动，每次移动一个网格单元
- 支持方向键（↑↓←→）和WASD键双重控制方案
- 蛇不能直接掉头（禁止向当前移动方向的反方向移动）
- 游戏开始时蛇长度为3个网格单元，初始位置在画布中央偏左，初始方向向右

#### 1.1.2 食物系统
- 食物在网格上随机生成，每次只存在一个食物
- 食物不能生成在蛇身体占据的网格位置上
- 蛇头移动到食物所在网格时判定为"吃到食物"
- 吃到食物后，蛇身长度增加1个网格单元
- 吃到食物后立即在新的随机位置生成下一个食物

#### 1.1.3 碰撞检测
- **墙壁碰撞**：蛇头超出画布边界（上、下、左、右任一方向）时游戏结束
- **自身碰撞**：蛇头与自身身体的任何一个网格单元重叠时游戏结束
- 碰撞发生后立即停止蛇的移动，进入游戏结束状态

#### 1.1.4 分数系统
- 每吃到一个食物，当前分数增加10分
- 游戏进行中实时显示当前分数
- 页面同时显示历史最高分
- 游戏结束时，若当前分数超过历史最高分，则更新最高分并持久化存储

#### 1.1.5 速度递增机制
- 游戏初始速度为150毫秒/步（蛇每150ms移动一格）
- 每吃到一个食物，移动间隔减少5毫秒（速度加快）
- 移动间隔最小值为50毫秒，达到后不再加速

### 1.2 游戏状态管理

#### 1.2.1 游戏状态定义
游戏共有4种状态，状态转换关系如下：
```
[等待开始] --(按空格键)--> [游戏中]
[游戏中] --(碰撞)--> [游戏结束]
[游戏中] --(按P键)--> [暂停中]
[暂停中] --(按P键)--> [游戏中]
[游戏结束] --(按空格键)--> [游戏中] (重新开始)
```

#### 1.2.2 各状态功能需求
- **开始界面**：显示游戏标题"贪吃蛇"和"按空格键开始游戏"提示
- **游戏进行中**：蛇自动持续移动，玩家可控制方向，实时更新画面
- **暂停功能**：P键暂停/继续，暂停时显示"游戏暂停"和"按P键继续"提示
- **游戏结束界面**：显示"游戏结束"、最终分数，若打破记录显示"新纪录!"，提示"按空格键重新开始"

### 1.3 键盘操作
| 按键 | 功能 | 生效状态 |
|------|------|---------|
| `ArrowUp` / `W` | 蛇向上移动 | 游戏中 |
| `ArrowDown` / `S` | 蛇向下移动 | 游戏中 |
| `ArrowLeft` / `A` | 蛇向左移动 | 游戏中 |
| `ArrowRight` / `D` | 蛇向右移动 | 游戏中 |
| `Space` | 开始游戏 / 重新开始 | 等待开始 / 游戏结束 |
| `P` | 暂停 / 继续 | 游戏中 / 暂停中 |

## 2. 技术实现要求

### 2.1 技术栈与交付物
- **技术栈**：HTML5 + CSS3 + JavaScript
- **交付物**：单个index.html文件，所有代码内嵌，无外部依赖
- **运行环境**：主流现代浏览器（Chrome、Firefox、Safari、Edge）

### 2.2 Canvas绘制要求
- 使用HTML5 `<canvas>`元素作为游戏主画布
- 画布尺寸：400px × 400px
- 网格规格：20×20网格，每个单元格20px×20px
- 每帧清空画布后重新绘制所有元素
- 绘制顺序：背景 → 网格线 → 食物 → 蛇 → 状态文字

### 2.3 游戏循环实现
- 使用`requestAnimationFrame`驱动渲染循环
- 采用时间差累积机制控制蛇的移动频率
- 仅在PLAYING状态下更新游戏逻辑，每帧都进行渲染

### 2.4 数据结构设计
```javascript
// 蛇的数据结构（数组表示，第一个元素为蛇头）
const snake = [
    { x: 5, y: 10 },  // 蛇头
    { x: 4, y: 10 },  // 蛇身
    { x: 3, y: 10 },  // 蛇尾
];

// 食物数据结构
let food = { x: 15, y: 10 };

// 游戏状态枚举
const GameState = {
    IDLE: 'IDLE',           // 等待开始
    PLAYING: 'PLAYING',     // 游戏中
    PAUSED: 'PAUSED',       // 暂停中
    GAME_OVER: 'GAME_OVER'  // 游戏结束
};
```

### 2.5 数据持久化
- 使用`localStorage`存储最高分
- 存储键名：`snakeGameHighScore`
- 页面加载时读取最高分，不存在则默认为0
- 游戏结束时更新最高分并写入`localStorage`
- 处理`localStorage`不可用的异常情况

### 2.6 键盘事件处理
- 使用`document.addEventListener('keydown', handler)`监听键盘事件
- 对方向键和空格键调用`event.preventDefault()`防止页面滚动
- 方向变更采用缓冲机制：按键时更新`nextDirection`，下次游戏逻辑更新时应用到`direction`
- 实现反方向检测逻辑

### 2.7 代码规范要求
- 所有JavaScript代码封装在立即执行函数(IIFE)或块级作用域内
- 使用`const`声明不可变变量，`let`声明可变变量，禁止使用`var`
- 关键函数和逻辑块需添加中文注释
- 函数职责单一，建议拆分为以下模块：
  - `init()` - 初始化/重置游戏状态
  - `gameLoop(timestamp)` - 游戏主循环
  - `update()` - 更新游戏逻辑
  - `render()` - 渲染画面
  - `drawGrid()` - 绘制背景网格线
  - `drawSnake()` - 绘制蛇
  - `drawFood()` - 绘制食物
  - `drawUI()` - 绘制状态文字
  - `generateFood()` - 生成随机食物位置
  - `handleKeydown(event)` - 键盘事件处理
  - `checkCollision()` - 碰撞检测
  - `getHighScore()` - 读取最高分
  - `setHighScore(score)` - 写入最高分

### 2.8 响应式设计
- 页面使用Flexbox布局实现水平垂直居中
- 设置`body { margin: 0; min-height: 100vh; }`确保占满视口
- 画布保持固定400×400尺寸，在小屏幕上通过页面滚动查看

## 4. 游戏规则说明

### 4.1 基本规则
1. **游戏目标**：控制蛇吃掉尽可能多的食物，获得高分
2. **控制方式**：使用方向键或WASD键控制蛇的移动方向
3. **得分规则**：每吃到一个食物得10分
4. **游戏结束条件**：
   - 蛇头撞到墙壁（画布边界）
   - 蛇头撞到自己的身体

### 4.2 游戏机制
1. **蛇的移动**：
   - 蛇在网格上连续移动，不能停止
   - 不能直接掉头（禁止向当前移动方向的反方向移动）
   - 每个游戏帧周期内只响应一次方向变更输入

2. **食物系统**：
   - 食物随机出现在网格上
   - 食物不会出现在蛇身体占据的位置
   - 吃到食物后蛇身长度增加1格

3. **速度机制**：
   - 初始速度：150毫秒/步
   - 每吃一个食物，移动间隔减少5毫秒
   - 最低速度限制：50毫秒/步

4. **游戏状态**：
   - **等待开始**：游戏初始状态，按空格键开始
   - **游戏中**：蛇自动移动，玩家可控制方向
   - **暂停中**：按P键暂停游戏，再次按P键继续
   - **游戏结束**：碰撞发生后进入此状态，按空格键重新开始

### 4.3 特殊规则
1. **最高分记录**：游戏记录历史最高分，刷新页面后仍然保留
2. **新纪录提示**：打破最高分时显示"新纪录!"提示
3. **速度限制**：速度有上限，防止过快导致无法操作

### 4.4 操作说明
```
控制方式：
  方向键 或 WASD键 - 控制蛇的移动方向
  空格键 - 开始游戏 / 重新开始游戏
  P键 - 暂停 / 继续游戏

游戏提示：
  1. 蛇不能直接掉头
  2. 分数越高，蛇移动越快
  3. 撞墙或撞到自己身体游戏结束
```

## 5. 开发优先级

### 5.1 优先级定义
- **P0（最高优先级）**：核心功能，必须实现
- **P1（高优先级）**：重要功能，建议实现
- **P2（中优先级）**：增强功能，可后续实现
- **P3（低优先级）**：优化功能，非必需

### 5.2 功能优先级划分

#### P0 - 核心功能（必须实现）
1. **基本游戏框架**
   - Canvas画布创建与初始化
   - 游戏循环基础结构
   - 基本页面布局

2. **蛇的基本功能**
   - 蛇的数据结构定义
   - 蛇的移动逻辑
   - 方向键控制（至少支持方向键）

3. **食物系统**
   - 食物随机生成
   - 吃食物检测
   - 蛇身增长逻辑

4. **碰撞检测**
   - 墙壁碰撞检测
   - 游戏结束逻辑

5. **基本UI**
   - 游戏开始界面
   - 游戏结束界面
   - 分数显示

#### P1 - 重要功能（建议实现）
1. **游戏状态管理**
   - 暂停/继续功能
   - 游戏状态流转

2. **增强控制**
   - WASD键支持
   - 禁止掉头逻辑
   - 输入缓冲机制

3. **分数系统**
   - 当前分数计算
   - 最高分持久化（localStorage）
   - 新纪录提示

4. **速度机制**
   - 速度递增逻辑
   - 速度上限控制

#### P2 - 增强功能（可后续实现）
1. **UI美化**
   - 网格线绘制
   - 蛇和食物的美化绘制（圆角、渐变等）
   - 画布发光效果

2. **操作体验优化**
   - 更流畅的动画效果
   - 食物生成动画
   - 碰撞特效

3. **额外功能**
   - 游戏难度选择
   - 音效支持
   - 触摸屏支持

#### P3 - 优化功能（非必需）
1. **高级特性**
   - 游戏回放功能
   - 排行榜系统
   - 成就系统

2. **跨平台优化**
   - 移动端适配优化
   - 离线PWA支持
   - 多语言支持

### 5.3 开发阶段建议

#### 第一阶段：核心功能实现（P0）
1. 搭建基础HTML结构和Canvas画布
2. 实现蛇的移动和方向控制
3. 实现食物生成和吃食物逻辑
4. 实现碰撞检测和游戏结束
5. 实现基本UI界面

#### 第二阶段：完善功能（P1）
1. 添加暂停/继续功能
2. 实现WASD键支持
3. 添加禁止掉头逻辑
4. 实现分数系统和最高分持久化
5. 添加速度递增机制

#### 第三阶段：UI美化（P2）
1. 美化蛇和食物的绘制
2. 添加网格线和画布效果
3. 优化文字显示和布局
4. 添加动画效果

#### 第四阶段：优化和扩展（P3）
1. 性能优化和代码重构
2. 添加额外功能（如音效、难度选择等）
3. 跨平台适配优化

### 5.4 验收标准
1. **基础验收**：所有P0功能正常工作，游戏可玩
2. **完整验收**：所有P0和P1功能实现，用户体验良好
3. **优秀验收**：所有P0、P1和P2功能实现，UI精美
4. **卓越验收**：所有功能实现，性能优秀，扩展性强
