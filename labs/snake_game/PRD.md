## 1. 项目概述

### 1.1 项目名称

**Snake Game** -- 经典贪吃蛇 HTML5 小游戏

### 1.2 项目目标

开发一款基于浏览器的经典贪吃蛇小游戏，具备现代化的视觉设计风格和流畅的游戏体验。玩家通过键盘控制蛇的移动方向，吃掉随机出现的食物来获取分数，蛇的身体会随之增长，难度逐步提升。游戏支持分数记录与最高分持久化存储，提供完整的开始、暂停、结束等游戏状态管理。

### 1.3 技术栈

| 技术       | 说明                                      |
| ---------- | ----------------------------------------- |
| HTML5      | 页面结构，使用 `<canvas>` 元素作为游戏画布 |
| CSS3       | 页面样式，内嵌于 `<style>` 标签中          |
| JavaScript | 游戏逻辑，内嵌于 `<script>` 标签中         |

> **交付物**: 单个 `index.html` 文件，所有 CSS 和 JavaScript 代码内嵌，无任何外部依赖。打开即可运行，无需构建工具或服务器。

### 1.4 目标用户

- 希望在浏览器中快速体验休闲游戏的用户
- 对前端游戏开发感兴趣的学习者

### 1.5 成功指标

- 游戏可在主流浏览器（Chrome、Firefox、Safari、Edge）中正常运行
- 游戏运行帧率稳定，无明显卡顿
- 所有功能需求均已实现并可正常使用

## 2. 功能需求

### 2.1 核心玩法

#### 2.1.1 蛇的移动

| 需求编号 | 描述 |
|---------|------|
| F-001 | 蛇在游戏画布的网格上移动，每次移动一个网格单元 |
| F-002 | 支持**方向键**（上 `ArrowUp`、下 `ArrowDown`、左 `ArrowLeft`、右 `ArrowRight`）控制蛇的移动方向 |
| F-003 | 同时支持 **WASD** 键控制（W 上、A 左、S 下、D 右） |
| F-004 | 蛇不能直接掉头（即向当前移动方向的反方向移动）。例如：蛇正在向右移动时，按左键无效 |
| F-005 | 游戏开始时，蛇的初始长度为 **3 个网格单元** |
| F-006 | 游戏开始时，蛇的初始位置在画布**中央偏左**区域，初始移动方向为**向右** |
| F-007 | 每个游戏帧周期内只响应**一次**方向变更输入，防止快速连按导致的穿身 bug |

#### 2.1.2 食物系统

| 需求编号 | 描述 |
|---------|------|
| F-008 | 食物在画布的网格上**随机**生成，每次只存在**一个**食物 |
| F-009 | 食物不能生成在蛇身体当前占据的网格位置上 |
| F-010 | 蛇头移动到食物所在网格时，判定为"吃到食物" |
| F-011 | 吃到食物后，蛇的身体长度**增加 1 个网格单元**（尾部增长） |
| F-012 | 吃到食物后，立即在新的随机位置生成下一个食物 |

#### 2.1.3 碰撞检测与游戏结束

| 需求编号 | 描述 |
|---------|------|
| F-013 | **墙壁碰撞**: 蛇头超出画布边界（上、下、左、右任一方向）时，游戏结束 |
| F-014 | **自身碰撞**: 蛇头与自身身体的任何一个网格单元重叠时，游戏结束 |
| F-015 | 游戏结束时，立即停止蛇的移动，进入游戏结束状态 |

#### 2.1.4 分数系统

| 需求编号 | 描述 |
|---------|------|
| F-016 | 每吃到一个食物，当前分数增加 **10 分** |
| F-017 | 游戏进行中，实时显示**当前分数** |
| F-018 | 页面上同时显示**历史最高分** |
| F-019 | 游戏结束时，若当前分数超过历史最高分，则更新最高分并持久化存储 |

#### 2.1.5 速度递增机制

| 需求编号 | 描述 |
|---------|------|
| F-020 | 游戏初始速度为 **150 毫秒/步**（即蛇每 150ms 移动一格） |
| F-021 | 每吃到一个食物，移动间隔减少 **5 毫秒**，即速度加快 |
| F-022 | 移动间隔的最小值为 **50 毫秒**，达到后不再加速（防止速度过快无法操作） |

> **速度计算公式**: `interval = Math.max(50, 150 - (score / 10) * 5)`

### 2.2 游戏状态管理

游戏共有 **4 种状态**，状态之间的转换关系如下:

```
[IDLE/等待开始] --(按空格键)--> [PLAYING/游戏中]
[PLAYING/游戏中] --(碰撞)--> [GAME_OVER/游戏结束]
[PLAYING/游戏中] --(按P键)--> [PAUSED/暂停中]
[PAUSED/暂停中] --(按P键)--> [PLAYING/游戏中]
[GAME_OVER/游戏结束] --(按空格键)--> [PLAYING/游戏中] (重新开始)
```

#### 2.2.1 开始界面 (IDLE)

| 需求编号 | 描述 |
|---------|------|
| F-023 | 游戏加载完成后，默认进入**开始界面** |
| F-024 | 开始界面在画布上显示游戏标题 **"贪吃蛇"** |
| F-025 | 开始界面显示提示文字 **"按空格键开始游戏"** |
| F-026 | 按下**空格键** (`Space`) 后，进入游戏进行状态 |

#### 2.2.2 游戏进行中 (PLAYING)

| 需求编号 | 描述 |
|---------|------|
| F-027 | 蛇按照当前方向自动持续移动 |
| F-028 | 玩家可通过键盘改变蛇的移动方向 |
| F-029 | 实时更新并渲染画面（蛇的位置、食物、分数） |

#### 2.2.3 暂停功能 (PAUSED)

| 需求编号 | 描述 |
|---------|------|
| F-030 | 游戏进行中按下 **P 键**，游戏暂停，蛇停止移动 |
| F-031 | 暂停时在画布上显示 **"游戏暂停"** 提示文字 |
| F-032 | 暂停时显示 **"按 P 键继续"** 提示文字 |
| F-033 | 暂停状态下再次按下 **P 键**，游戏恢复运行 |
| F-034 | 暂停状态下，方向键和 WASD 键输入**不生效** |

#### 2.2.4 游戏结束界面 (GAME_OVER)

| 需求编号 | 描述 |
|---------|------|
| F-035 | 游戏结束时在画布上显示 **"游戏结束"** 文字 |
| F-036 | 显示**最终分数**，格式为 **"得分: XXX"** |
| F-037 | 若打破最高分记录，额外显示 **"新纪录!"** 提示 |
| F-038 | 显示提示文字 **"按空格键重新开始"** |
| F-039 | 按下**空格键**后，重置所有游戏数据（蛇的位置、长度、分数、速度），重新开始游戏 |

### 2.3 键盘操作汇总

| 按键 | 功能 | 生效状态 |
|------|------|---------|
| `ArrowUp` / `W` | 蛇向上移动 | PLAYING |
| `ArrowDown` / `S` | 蛇向下移动 | PLAYING |
| `ArrowLeft` / `A` | 蛇向左移动 | PLAYING |
| `ArrowRight` / `D` | 蛇向右移动 | PLAYING |
| `Space` | 开始游戏 / 重新开始 | IDLE / GAME_OVER |
| `P` | 暂停 / 继续 | PLAYING / PAUSED |

## 4. 技术要求

### 4.1 文件结构

项目交付为**单个 `index.html` 文件**，内部结构如下:

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>贪吃蛇</title>
    <style>
        /* 所有 CSS 样式内嵌于此 */
    </style>
</head>
<body>
    <!-- HTML 结构 -->
    <script>
        /* 所有 JavaScript 代码内嵌于此 */
    </script>
</body>
</html>
```

### 4.2 Canvas 绘制

| 要求编号 | 描述 |
|---------|------|
| T-001 | 使用 HTML5 `<canvas>` 元素作为游戏主画布 |
| T-002 | 通过 `canvas.getContext('2d')` 获取 2D 绘图上下文 |
| T-003 | 每帧清空画布后重新绘制所有元素（背景 -> 网格线 -> 食物 -> 蛇 -> 状态文字） |
| T-004 | 绘制顺序需确保视觉层级正确（蛇在食物和网格线之上，状态文字覆盖层在最上方） |

### 4.3 游戏循环

| 要求编号 | 描述 |
|---------|------|
| T-005 | 使用 `requestAnimationFrame` 驱动渲染循环，确保与浏览器刷新率同步 |
| T-006 | 通过**时间差累积**机制控制蛇的移动频率（而非 `setInterval`），实现精确的速度控制 |
| T-007 | 游戏循环伪代码如下: |

```javascript
// 游戏循环伪代码
let lastMoveTime = 0;

function gameLoop(currentTime) {
    requestAnimationFrame(gameLoop);

    // 仅在 PLAYING 状态下更新游戏逻辑
    if (gameState === 'PLAYING') {
        if (currentTime - lastMoveTime >= moveInterval) {
            updateSnakePosition(); // 更新蛇的位置
            checkCollision();       // 碰撞检测
            checkFood();            // 食物检测
            lastMoveTime = currentTime;
        }
    }

    render(); // 每帧都进行渲染
}
```

### 4.4 数据结构设计

#### 蛇的数据结构

```javascript
// 蛇使用数组表示，每个元素为一个网格坐标对象
// 数组第一个元素为蛇头，最后一个元素为蛇尾
const snake = [
    { x: 5, y: 10 },  // 蛇头
    { x: 4, y: 10 },  // 蛇身
    { x: 3, y: 10 },  // 蛇尾
];

// 当前移动方向
let direction = { x: 1, y: 0 }; // 向右

// 下一帧待执行的方向（防止单帧内多次转向）
let nextDirection = { x: 1, y: 0 };
```

#### 食物的数据结构

```javascript
// 食物为一个网格坐标对象
let food = { x: 15, y: 10 };
```

#### 游戏状态

```javascript
// 游戏状态枚举
const GameState = {
    IDLE: 'IDLE',           // 等待开始
    PLAYING: 'PLAYING',     // 游戏中
    PAUSED: 'PAUSED',       // 暂停中
    GAME_OVER: 'GAME_OVER'  // 游戏结束
};

let gameState = GameState.IDLE;
```

### 4.5 最高分持久化

| 要求编号 | 描述 |
|---------|------|
| T-008 | 使用 `localStorage` 存储最高分 |
| T-009 | 存储键名: `snakeGameHighScore` |
| T-010 | 页面加载时从 `localStorage` 读取最高分，若不存在则默认为 0 |
| T-011 | 游戏结束时，若当前分数大于最高分，将新最高分写入 `localStorage` |
| T-012 | 需处理 `localStorage` 不可用的异常情况（如隐私模式），使用 `try-catch` 包裹 |

### 4.6 键盘事件处理

| 要求编号 | 描述 |
|---------|------|
| T-013 | 使用 `document.addEventListener('keydown', handler)` 监听键盘事件 |
| T-014 | 对方向键和空格键调用 `event.preventDefault()`，防止页面滚动 |
| T-015 | 方向变更采用**缓冲机制**: 玩家按键时仅更新 `nextDirection`，在下一次游戏逻辑更新时才将 `nextDirection` 应用到 `direction` |
| T-016 | 反方向检测逻辑: 新方向与当前方向之和若为零向量，则忽略该输入 |

### 4.7 食物生成算法

```javascript
// 食物生成伪代码
function generateFood() {
    let newFood;
    do {
        newFood = {
            x: Math.floor(Math.random() * gridCount), // 0 ~ 19
            y: Math.floor(Math.random() * gridCount)   // 0 ~ 19
        };
    } while (isOnSnake(newFood)); // 确保不在蛇身上
    return newFood;
}
```

### 4.8 代码规范

| 要求编号 | 描述 |
|---------|------|
| T-017 | 所有 JavaScript 代码封装在一个**立即执行函数 (IIFE)** 或在 `<script>` 标签内使用块级作用域，避免全局变量污染 |
| T-018 | 使用 `const` 声明不可变变量，`let` 声明可变变量，**禁止使用 `var`** |
| T-019 | 关键函数和逻辑块需添加**中文注释** |
| T-020 | 函数职责单一，建议拆分为以下函数模块: |

| 函数名 | 职责 |
|--------|------|
| `init()` | 初始化/重置游戏状态 |
| `gameLoop(timestamp)` | 游戏主循环 |
| `update()` | 更新游戏逻辑（蛇移动、碰撞检测、食物检测） |
| `render()` | 渲染画面（清屏、绘制网格、绘制蛇、绘制食物、绘制 UI） |
| `drawGrid()` | 绘制背景网格线 |
| `drawSnake()` | 绘制蛇 |
| `drawFood()` | 绘制食物 |
| `drawUI()` | 绘制状态文字（开始、暂停、结束等） |
| `generateFood()` | 生成随机食物位置 |
| `handleKeydown(event)` | 键盘事件处理 |
| `checkCollision()` | 碰撞检测（墙壁 + 自身） |
| `getHighScore()` | 从 localStorage 读取最高分 |
| `setHighScore(score)` | 将最高分写入 localStorage |
